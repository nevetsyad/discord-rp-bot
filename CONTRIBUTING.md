# Contributing to Discord RP Bot

Thank you for your interest in contributing to the Discord RP Bot project! This document provides guidelines for contributing.

## How to Contribute

### Reporting Bugs
- Use the GitHub Issues system to report bugs
- Include detailed steps to reproduce the issue
- Include your environment information (Node.js version, Discord.js version, etc.)

### Suggesting Features
- Open an issue with a clear description of the feature
- Explain how the feature would improve the bot
- Provide examples of use cases

### Submitting Pull Requests
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test thoroughly
5. Commit your changes (`git commit -m 'Add amazing feature'`)
6. Push to the branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

## Development Guidelines

### Code Style
- Use consistent JavaScript formatting
- Follow the existing code patterns
- Use meaningful variable and function names
- Add comments for complex logic

### Testing
- Test your changes before submitting
- Ensure existing functionality still works
- Test on different Discord.js versions if applicable

### Database Changes
- If making database changes, update the models
- Ensure migrations are handled properly
- Test with a fresh database if needed

### Documentation
- Update README.md if adding new commands
- Add comments for new features
- Update this contributing guide if adding new processes

## Getting Started

1. Clone the repository: `git clone <repository-url>`
2. Install dependencies: `npm install`
3. Set up environment variables (see `.env.example`)
4. Run the setup script: `npm run setup`
5. Start the bot: `npm start`

## Support

If you need help:
- Check the existing issues and documentation
- Open a new issue with detailed information
- Join our Discord server for community support

Thank you for contributing!
